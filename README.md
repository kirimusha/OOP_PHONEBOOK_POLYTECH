**1. Contact**

**Назначение:** Основной класс, представляющий контакт в телефонной книге.  
**Описание:**  
Хранит все данные контакта:
- Личные данные (имя, фамилия, отчество)
- Контактную информацию (адрес, email, телефоны)
- Дату рождения

**Основные функции:**
- Валидация данных контакта
- Сериализация/десериализация в JSON
- Управление телефонами (добавление, удаление)
- Перегрузка операторов сравнения



**2. PhoneNumber**

**Назначение:** Класс для представления и работы с номером телефона.  
**Описание:**  
Хранит номер телефона и его тип (мобильный, домашний, рабочий).

**Основные функции:**
- Валидация формата номера
- Нормализация номера (приведение к стандартному виду)
- Преобразование типа телефона в строку
- Сериализация в JSON



**3. ContactManager**

**Назначение:** Менеджер для управления контактами через репозиторий.  
**Описание:**  
Служит промежуточным слоем между UI и хранилищем данных.

**Основные функции:**
- Добавление/удаление/обновление контактов
- Поиск контактов по имени
- Получение всех контактов
- Массовое обновление контактов



**4. IContactRepository**

**Назначение:** Абстрактный интерфейс репозитория контактов.  
**Описание:**  
Определяет контракт для любых реализаций хранилищ контактов.

**Основные методы:**
- `addContact()`, `removeContact()`, `updateContact()`
- `getContact()`, `getAllContacts()`



**5. FileRepository**

**Назначение:** Реализация репозитория для хранения контактов в JSON-файле.  
**Описание:**  
Работает с файловой системой, сохраняет и загружает контакты в формате JSON.

**Основные функции:**
- Загрузка контактов из файла
- Сохранение контактов в файл
- Работа с коллекцией контактов в памяти



**6. Validators**

**Назначение:** Класс утилит для валидации различных типов данных.  
**Описание:**  
Содержит статические методы для проверки корректности данных.

**Проверяемые данные:**
- Имена (имя, фамилия, отчество)
- Email-адреса (с обязательным содержанием имени)
- Номера телефонов
- Даты рождения



**7. UI (User Interface)**

**Назначение:** файл с функциями для реализации пользовательского интерфейса в консоли.  
**Описание:**  
Содержит функции для взаимодействия с пользователем через меню.

**Основные функции:**
- Отображение главного меню
- Просмотр всех контактов
- Добавление/редактирование/удаление контактов
- Поиск и сортировка контактов
- Вспомогательные функции (очистка экрана, ожидание ввода)


**8. PhoneType**

**Назначение:** Перечисление (enum) для типов телефонов.  
**Значения:** `Mobile`, `Home`, `Work`


### **Архитектурная схема:**
```
Пользователь 
    ↓
UI (консольный интерфейс)
    ↓
ContactManager (бизнес-логика)
    ↓
IContactRepository ←─ FileRepository (хранилище)
    ↓
Contact (модель данных) ←─ PhoneNumber (компонент)
    ↓
Validators (валидация данных)
```
